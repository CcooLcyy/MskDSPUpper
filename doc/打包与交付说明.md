# 打包与交付说明

## 目标

- 用统一脚本把 `Tauri NSIS` 产物收敛为可分发、可校验、可排障的交付物。
- 为现场部署保留安装、启动、停止、升级、版本检查、日志探测等最小运维动作。

## 范围与非目标

### 范围

- `package/staging` 和 `package/out`
- 交付 zip、symbols zip、校验文件
- PowerShell 安装与运维脚本

### 非目标

- 不生成 Docker 镜像
- 不生成 Linux/macOS 包

## 输入输出

### 输入

- `dist/`
- `src-tauri/target/**/release/bundle/`
- `src-tauri/target/**/release/*.pdb`
- `scripts/workflow/templates/*.ps1`

### 输出

- `package/out/<channel>/<platform>/<artifact>.zip`
- `package/out/<channel>/<platform>/<artifact>-symbols.zip`
- `package/out/<channel>/<platform>/<artifact>-SHA256SUMS.txt`

## 生成方式

1. 生成构建元数据：

```bash
node scripts/workflow/emit-build-metadata.mjs --channel nightly
```

2. 为当前渠道应用预发布版本：

```bash
node scripts/workflow/apply-channel-version.mjs --metadata-file package/build-metadata.json
```

3. 渲染 CI 专用 Tauri 配置：

```bash
node scripts/workflow/render-tauri-config.mjs --metadata-file package/build-metadata.json
```

4. 产出 Tauri 安装包：

```bash
npx tauri build --bundles nsis --target x86_64-pc-windows-msvc --ci --config package/generated/tauri.ci.conf.json
```

5. 收集到统一 staging 目录：

```bash
node scripts/workflow/stage-release.mjs --metadata-file package/build-metadata.json
```

6. 生成交付包与 checksum：

```bash
node scripts/workflow/create-delivery-bundle.mjs --metadata-file package/build-metadata.json
node scripts/workflow/write-sha256sums.mjs --metadata-file package/build-metadata.json
```

## 部署

- 方式一：
  - 解压交付 zip
  - 运行 `install.ps1`
- 方式二：
  - 直接运行交付目录中的安装包

## 启动

```powershell
.\start.ps1
```

## 停止

```powershell
.\stop.ps1
```

## 升级

```powershell
.\upgrade.ps1
```

## 版本校验

```powershell
.\version.ps1
```

## 日志查看

```powershell
.\logs.ps1
```

- 当前仓库已经补齐 workflow / packaging 侧日志与 diagnostics。
- 运行时日志目录会优先探测：
  - 安装目录下 `logs`
  - `%LOCALAPPDATA%\MskDSP Upper\logs`
  - `%APPDATA%\MskDSP Upper\logs`

## 回滚 / 替换旧包

1. 先执行 `stop.ps1`
2. 备份旧版交付包与 `SHA256SUMS`
3. 使用新版本 `install.ps1` 或 `upgrade.ps1`
4. 若回退，重新运行旧版本交付包中的 `install.ps1`

## 约束

- 所有最终可交付文件都必须先进入 `package/staging`，再进入 `package/out`。
- 所有发布渠道都必须带 checksum 文件。
- symbols 包与主包需要同名基底，便于问题追踪。
