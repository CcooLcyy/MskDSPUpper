# 下位机更新上传与提权修复方案

## 1. 背景与问题

当前下位机更新流程把 SSH 登录、文件上传和需要 root 权限的操作统一包装为
`sudo -n -- sh -c`。普通账号能够通过 SSH 密码登录，但未配置免密 sudo 时，远端会立即返回
`sudo: a password is required`。密码上传实现此时仍继续发送完整安装包，且直到发送结束后才读取远端退出状态；SSH 接收窗口耗尽后，上传可能长期等待且没有成功或失败日志。

现场日志还显示，更新包已在上位机下载并通过 SHA-256 校验，因此本次修复聚焦上位机到下位机的 SSH 交付链路。

## 2. 目标

- 文件上传使用 SSH 登录用户权限，不再要求 sudo。
- 部署开始前验证安装目录可写、磁盘空间充足和 sudo 凭据有效。
- 运行镜像查询与安装支持单独的 sudo 密码，不假设 SSH 密码与 sudo 密码相同。
- sudo 密码只通过 SSH 加密通道的标准输入发送，不拼接到命令、错误或日志中。
- 远端命令提前退出时立即停止上传并返回远端错误。
- 上传具备无进度超时和整体超时，任何失败都有明确终态日志。
- 失败上传使用临时文件，远端命令退出时自动清理临时文件。

## 3. 非目标

- 本次不修改下位机 MskDSP、ModuleManager 或业务模块。
- 本次不自动修改目标机 sudoers，也不授予 `NOPASSWD: ALL`。
- 本次不引入或部署下位机 root 更新辅助程序。
- 本次不把 sudo 密码持久化到配置、文件或系统凭据库。
- 本次不改变更新清单、安装包格式和 SHA-256 校验规则。

## 4. 设计

### 4.1 认证与权限分层

SSH 认证继续使用现有密码或证书方式。上传请求、运行镜像查询请求和安装请求增加独立的
`sudo_password` 字段。

前端默认启用“sudo 密码与 SSH 密码相同”。关闭后显示独立的 sudo 密码输入框。独立密码只保存在当前页面内存中，不复用现有 SSH 密码保存接口。

需要 root 权限的查询、预检和安装命令使用 `sudo -S -p '' -- sh -c ...`。sudo 密码通过 SSH Channel 标准输入发送，发送后关闭输入流；日志只记录认证方式和执行阶段。

### 4.2 上传预检

上传开始前执行两类远端检查：

1. 以 SSH 登录用户创建或确认安装目录，验证目录可写，并读取可用空间。
2. 使用 sudo 密码执行无副作用的 `true`，验证后续安装能够提权。

目录不可写、空间小于安装包大小或 sudo 验证失败时，上传不发送任何安装包数据并直接返回中文错误。

### 4.3 普通用户上传

上传命令不再通过 `build_root_shell_command` 包装。远端以登录用户执行：创建安装目录、删除旧临时文件、将标准输入写入 `<package>.uploading`、原子重命名到正式文件并增加执行权限。

远端 shell 使用 trap 在命令异常退出时删除 `.uploading` 文件，避免失败上传长期占用空间。

### 4.4 远端退出与超时

上传发送过程必须持续消费 SSH Channel 消息，并关注 stderr、`ExitStatus`、`Eof` 和 `Close`。远端提前退出或关闭时，发送任务立即停止并返回已收集的远端错误，不能等本地文件全部读完。

上传设置两层限制：

- 连续 60 秒没有新增已上传字节时返回“上传无进度超时”。
- 整个上传流程超过 30 分钟时返回“上传整体超时”。

外层命令负责记录失败阶段、已上传字节、包大小和耗时，不记录任何密码。

### 4.5 安装与运行镜像查询

运行镜像查询和安装继续执行现有 root 命令，但改用独立 sudo 密码。SSH 密码认证路径和证书认证路径都必须把 sudo 密码写入远端命令标准输入。

安装命令执行结束后保留现有容器恢复与镜像 ID 校验流程。`17000` 端口在安装前不可达不阻止更新，但安装后运行镜像校验失败必须继续显示为部署失败。

## 5. 异常场景

- SSH 登录失败：返回现有 SSH 认证错误，不执行预检和上传。
- sudo 密码为空或错误：预检失败，不发送安装包数据。
- 安装目录不可写：预检失败并显示目标目录。
- 磁盘空间不足：预检失败并显示需要和可用字节数。
- 远端命令在上传中退出：立即停止发送，返回退出码和 stderr。
- SSH 连接中断：返回中断时已上传字节数，远端 trap 清理临时文件。
- 上传无进度或超过整体时限：关闭 SSH 会话并返回对应超时错误。
- 安装失败：保留退出码、stdout 和 stderr，进入现有安装失败状态。

## 6. 测试与验收标准

- 单元测试证明上传命令不包含 sudo，root 查询和安装命令使用 `sudo -S`。
- 单元测试证明 sudo 密码不会出现在构造后的远端命令中。
- 远端命令立即报错时，上传在数秒内失败且不会读取完整测试包。
- 目录不可写、空间不足和 sudo 密码错误时，上传字节数保持为零。
- 模拟无窗口更新或无上传进度时，上传返回无进度超时。
- 正常上传完成后远端只保留正式文件，不保留 `.uploading` 文件。
- 前端在“密码相同”开启时向三类请求传递 SSH 密码，关闭时传递独立 sudo 密码。
- 前端独立 sudo 密码不调用 SSH 密码保存接口，也不显示在部署结果中。
- 日志对预检、上传、安装均有开始、成功或失败终态，且不包含 SSH 或 sudo 明文密码。
