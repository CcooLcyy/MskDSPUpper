# DLT645 点位校验与三协议复制交互方案

## 目标

补齐 DLT645 上位机点位编辑和复制时的前置校验，并统一 IEC104、ModbusRTU、DLT645 的复制点位交互，使用户在点击保存前即可发现点位 Tag 或协议地址冲突。

## 范围

- 三种协议的复制点位都打开点位弹窗，复制结果先带出原 Tag，不自动追加 `_copy`。
- 新增模式下原 Tag 立即显示重复校验，用户修改为唯一 Tag 后才能保存；编辑当前点位时允许保留原 Tag。
- IEC104 复制继续自动选择下一个可用 IOA，ModbusRTU 复制继续自动选择下一个可用地址。
- 普通点位不能与其他点位复用同一 DI。
- BOOL bit 点允许在同一 DI 下复用不同 byte/bit，但相同 byte/bit 不能重复；同一 DI 下的 data_len 必须一致。
- 不改变数据块与单点之间已有的兼容规则，也不自动推导或递增 DLT645 DI。
- 连接复制仍使用连接名称的 `_copy` 规则，不改变点位 Tag。

## 验收标准

- Tag 重复时，上位机表单显示“标签已存在”，不会发起保存请求。
- DI 冲突时，上位机表单显示对应的 DI 冲突提示，不依赖下位机返回错误。
- 三种协议复制 `temperature` 都先显示 `temperature` 和重复提示，修改后才能保存。
- 现有三种协议的点位新增、编辑、删除、地址分配和连接复制流程保持不变。

## 测试

- 为命名生成和点位冲突判断补充纯逻辑测试。
- 为三个协议页面补充源代码级工作流断言，确认复制入口打开弹窗并使用重复校验。
