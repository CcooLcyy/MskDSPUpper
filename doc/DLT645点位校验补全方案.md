# DLT645 点位校验补全方案

## 目标

补齐 DLT645 上位机点位编辑和复制时的前置校验，使用户在点击保存前即可发现点位 Tag 或 DI 冲突，并让复制点位的 Tag 命名与 IEC104、ModbusRTU 保持一致。

## 范围

- 点位 Tag 在当前连接的单点列表内必须唯一，编辑当前点位时允许保留原 Tag。
- 复制点位时统一生成 `原 Tag_copy`、`原 Tag_copy_2`、`原 Tag_copy_3`，复制已有副本时不生成 `copy_copy`。
- 普通点位不能与其他点位复用同一 DI。
- BOOL bit 点允许在同一 DI 下复用不同 byte/bit，但相同 byte/bit 不能重复；同一 DI 下的 data_len 必须一致。
- 不改变数据块与单点之间已有的兼容规则，也不自动推导或递增 DLT645 DI。

## 验收标准

- Tag 重复时，上位机表单显示“标签已存在”，不会发起保存请求。
- DI 冲突时，上位机表单显示对应的 DI 冲突提示，不依赖下位机返回错误。
- 复制 `temperature` 得到 `temperature_copy`；复制 `temperature_copy` 时得到下一个可用的 `temperature_copy_N`。
- 现有 DLT645 点位新增、编辑、删除和连接复制流程保持不变。

## 测试

- 为命名生成和点位冲突判断补充纯逻辑测试。
- 为 DLT645 页面补充源代码级工作流断言，确认表单使用 Tag/DI 重复校验。
