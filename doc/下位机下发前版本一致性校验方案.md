# 下位机下发前版本一致性校验方案

## 目标

在上位机执行“下发并安装”前，重新确认目标下位机是否已经运行待下发构建，避免重复上传和重复安装。

## 判定规则

- 以更新清单中的 Docker `image_id` 作为构建身份，不使用展示用的 `version` 或 `package_version` 做设备侧比较。
- 下发前查询目标机 `mskdsp` 容器的运行状态和 `image_id`。
- 目标容器正在运行且 `image_id` 与清单一致时，标记为“已是最新”，跳过上传和安装。
- 目标机无法确认、容器不存在、容器未运行或镜像 ID 无效时，不在上位机直接判定为最新。
- 镜像 ID 不一致时继续上传和安装；安装完成后仍保留镜像 ID 复核。

## 一致性边界

前端在开始上传前执行一次检查，用于及时反馈并避免无效传输；安装接口同时携带期望的 `image_id`，在真正执行安装命令前再次检查，防止设备在前端检查后发生变化或绕过前端直接调用接口。
