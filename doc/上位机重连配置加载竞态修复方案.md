# 上位机重连配置加载竞态修复方案

## 1. 背景与问题

现场上位机日志显示，ModuleManager 的 gRPC Channel 已建立后，首页立即并发请求 IEC104、ModbusRTU 等模块配置；此时 `GetRunningModuleInfo` 尚未返回，模块名到外部 gRPC 地址的缓存仍为空，导致配置读取报“模块地址未知，请先刷新运行信息”。数毫秒后地址缓存刷新完成，但首次失败的配置读取不会自动补偿。

同一份日志还显示，相同 ModuleManager 地址会被重复设置并反复清理 Channel。现有命令会同时停止协议实时数据流，因此普通页面初始化也可能中断已经工作的订阅。

## 2. 目标

- ModuleManager 地址设置完成后，必须先成功刷新运行模块地址，再并发读取各模块配置。
- 相同地址的普通初始化必须幂等，不清理 Channel、模块地址缓存或实时订阅。
- 用户主动点击“连接”时执行强制重连：停止旧订阅、清理运行时缓存、刷新模块地址，并重新启动实时订阅任务。
- 跨越地址切换或强制重连边界的旧运行模块响应不得覆盖新一代地址缓存。
- 日志能够区分地址未变化、地址发生变化、强制重连，以及订阅建立、结束和重试。
- 配置读取失败时继续保留分区失败提示，不把失败误报为成功加载的空配置。

## 3. 非目标

- 不修改下位机协议、ModuleManager 或各业务模块的 gRPC 接口。
- 不对保存配置、启停模块等有副作用的 RPC 自动重试。
- 本次不引入新的全局状态管理框架，也不调整页面信息架构。
- 本次不处理与启动竞态无关的 DLT645、ModbusRTU 设备通信状态。

## 4. 设计

### 4.1 首页加载顺序

首页刷新分成两个阶段：

1. 设置 ModuleManager 地址并读取运行模块信息，完成模块地址缓存刷新。
2. 仅在运行模块信息成功后，并发读取模块清单以及 IEC104、ModbusRTU、DLT645、AGC 和 DataCenter 配置摘要。

运行模块信息作为唯一前置请求，避免模块清单与运行状态在空缓存下同时创建重复的 ModuleManager Channel。若运行模块信息失败，第二阶段不发出模块配置请求，并把相关分区标记为加载失败；模块清单仍可单独尝试读取，以保留尽可能多的诊断信息。

### 4.2 地址设置与强制重连

`set_manager_addr` 增加可选的强制重连语义：

- 地址变化：停止旧订阅，更新地址并清理 Channel 与模块地址缓存。
- 地址不变且非强制：保留现有 Channel、模块地址缓存和订阅。
- 地址不变且强制：停止旧订阅并清理运行时缓存，用于用户主动重连或下位机重启后的恢复。

每次清理运行时缓存都会递增连接代次。`GetRunningModuleInfo` 发起时记录当前代次，返回后仅在代次未变化时更新模块地址；过期响应返回明确错误并丢弃。

前端页面自动初始化使用非强制模式；“连接”按钮使用强制模式。

### 4.3 实时订阅恢复

协议实时数据流启动命令只负责确保后台任务存在，具体的影子配置同步和 Subscribe 由后台循环完成，避免命令路径和后台任务重复执行同一轮同步。

强制重连在运行模块信息刷新完成后重新启动该后台任务。若刷新失败，也要恢复后台任务并让其按退避策略继续尝试，但连接按钮仍显示失败。订阅日志至少包含任务启动、订阅建立、服务端结束、读取失败和重试等待。

### 4.4 日志

新增或调整以下日志字段：

- ModuleManager 地址、是否变化、是否强制重连、是否清理运行时缓存。
- 刷新后的模块名到 gRPC 地址映射。
- 实时流任务是否新建或已经运行。
- Subscribe 对应的 `conn_id`、建立结果、结束原因与重试等待秒数。

## 5. 异常场景

- ModuleManager 不可达：连接动作失败且不显示连接成功；配置读取不启动，实时订阅后台任务恢复后按退避策略继续探测。
- `GetRunningModuleInfo` 失败：不请求依赖模块地址的配置，首页提示运行状态及配置分区加载失败。
- 部分业务模块不可达：其余配置分区继续加载，失败分区单独提示。
- 实时订阅失败或结束：后台任务按现有指数退避策略重试，不阻塞配置页面。
- 下位机重启且动态模块端口变化：用户强制重连会清除旧模块地址，重新获取最新地址。
- 旧运行模块请求在重连后返回：连接代次不匹配，响应被丢弃且不得覆盖新地址。

## 6. 验收标准

- 在运行模块信息 Promise 完成前，首页不会调用任何业务模块配置读取 API。
- 运行模块信息成功后，各配置读取 API 可以并发执行并正确统计结果。
- 同地址非强制设置返回“未变化”，模块地址缓存保持不变。
- 强制重连会清除模块地址缓存；下一次配置读取只能使用重新获取的地址。
- 用户点击连接时，在模块信息刷新后调用实时数据流启动命令，再显示连接成功。
- 刷新失败时连接动作返回失败，但实时数据流后台任务会恢复并继续重试。
- 上位机日志可直接观察实时订阅建立或失败，不再只有“开始启动”而没有后续状态。
